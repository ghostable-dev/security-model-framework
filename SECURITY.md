# Security Policy

This repository contains documentation templates and examples. It does not ship application runtime code.

## Reporting A Vulnerability

If you find a security issue in this project, please report it privately.

- Use GitHub's private vulnerability reporting if it is enabled for this repository.
- If private reporting is not enabled, contact the maintainer privately before opening a public issue.
- Please include affected files, a clear description, and suggested remediation if available.

Do not open a public issue for an unpatched vulnerability.

## Template Security Guidance

If you believe one of the templates gives unsafe, incomplete, or misleading security guidance, please open an issue or pull request unless the report includes a sensitive exploit path.

## Application Security Models

This project promotes `SECURITY_MODEL.md` for application-level security behavior. GitHub's standard `SECURITY.md` should still be used for vulnerability reporting and disclosure policy.
